# パスノート 設計資料

このフォルダーには、パスノートの仕組みを理解し、安全に変更するための資料をまとめます。

フロントエンドに詳しくなくても全体像から順番に理解できることと、変更したい機能から関係するコードを逆引きできることを目標にしています。

## 最初に読む順番

1. [全体設計](overview.md)
   - 何を実現するツールか
   - 何を大切にしているか
   - 主要な機能と責務
2. [アーキテクチャ](architecture.md)
   - `data.js`、localStorage、画面の関係
   - 起動、変更、書き出しの流れ
3. [path-note.html 構成マップ](path-note-html-map.md)
   - 巨大なHTMLのどこに何があるか
   - 機能を変更するときに見る場所

## 資料の役割

| 資料 | 答える問い |
| --- | --- |
| 全体設計 | このツールは何を目指し、どのような方針で作られているか |
| アーキテクチャ | データと画面はどのようにつながっているか |
| HTML構成マップ | 変更したい機能はコードのどこにあるか |

## コードとの関係

配布時に利用者が必要とする製品ファイルは、次の2つです。

- `path-note.html`: 画面、スタイル、アプリ処理
- `data.js`: 初回表示データと、持ち運び用データ

設計資料は製品ファイルではありません。理解と保守のためにリポジトリ内へ置いています。

## 資料を更新するタイミング

次の変更を行った場合は、関係する資料も確認します。

- データの保存方法や形式を変えた
- `path-note.html` に新しい責務の章を追加した
- 主要機能や主要操作を追加・削除した
- 製品ファイルを2ファイルに保つ方針を変えた
- 保守時に「なぜこの実装なのか」が分かりにくかった
