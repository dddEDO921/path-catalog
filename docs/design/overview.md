# 全体設計

## パスノートとは

パスノートは、ファイルパス、フォルダーパス、URLを分類して、必要な場所をすぐに見つけてコピーするためのツールです。

ブラウザで開くだけで使えます。パスの追加・編集・削除・検索・分類整理もブラウザ上で行います。

## 解決したい問題

- よく使うパスがチャット、メモ、共有フォルダーなどへ散らばる
- パスを覚えておくことが難しい
- 大量のパスをフラットな一覧だけで管理することが難しい
- 専用サーバーやインストール作業なしで使いたい
- ブラウザで登録したデータを、ファイルとして持ち運びたい

## 想定する使い方

1. `path-note.html`をブラウザで開く
2. タブ、カテゴリ、サブカテゴリから目的のパスを探す
3. 一覧のパスをクリックしてコピーする
4. 必要に応じて検索、追加、編集、削除を行う
5. データを残すときは`data.js`へ書き出す

## 大切にする設計方針

### 利用者はブラウザを中心に操作する

`data.js`を直接編集することもできますが、通常はブラウザ上で登録・編集します。データ形式の知識がなくても使えることを優先します。

### 製品ファイルを2つに保つ

配布時に必要なファイルは、`path-note.html`と`data.js`だけです。

JavaScriptやCSSを別ファイルへ分割するとコードは追いやすくなりますが、配布・コピー・利用開始が複雑になります。パスノートでは、コードの分割よりも利用開始の簡単さを優先します。

### ブラウザ内のデータを日常利用の正本にする

初回表示では`data.js`を読み込みます。その後の変更はlocalStorageへ保存され、次回以降はlocalStorageから表示します。

永続化や別環境への移行が必要な場合は、ブラウザから新しい`data.js`を書き出します。

### 大量のパスを扱っても管理負荷を増やさない

タブ・カテゴリ・サブカテゴリによる分類と横断検索を中心にします。利用者が大量の個別設定を管理し続ける機能は慎重に採用します。

## 製品の構成

| ファイル | 役割 |
| --- | --- |
| `path-note.html` | 画面構造、見た目、データ操作、イベント処理、インポート・エクスポート |
| `data.js` | 初回表示データ、バックアップ、別環境への移行 |

補助ファイルとして、README画像、ロゴ、スモークチェック、設計資料があります。これらはツールの利用時には必須ではありません。

## 主要機能

| 分類 | 機能 |
| --- | --- |
| 探す | タブ・カテゴリ絞り込み、サブカテゴリ折りたたみ、横断検索 |
| 使う | パスのコピー、ノート表示、詳細表示 |
| 登録する | パス、タブ、カテゴリ、サブカテゴリの追加 |
| 整理する | パスの編集・削除、分類の削除 |
| 表示する | スタンダード、オープン、コンパクトの表示モード |
| 持ち運ぶ | `data.js`の読み込み・書き出し |

## データに関する責任分担

| データ | 保存場所 | 役割 |
| --- | --- | --- |
| カタログ | localStorage | 日常利用中の最新データ |
| 初期カタログ | `data.js` | localStorageがない場合の初期値 |
| 書き出しデータ | ダウンロードした`data.js` | バックアップ、配布、移行 |
| 表示モード | localStorage | 利用者ごとの見た目 |
| 折りたたみ状態 | localStorage | 利用者ごとの表示状態 |

## 現在の制約

- データは利用中のブラウザに保存されるため、ブラウザデータを削除すると消える
- ブラウザ間や端末間では自動同期されない
- 同時編集や複数利用者向けの権限管理はない
- CSSとフォントはCDNを利用するため、完全なオフライン表示では見た目が崩れる可能性がある
- 単一HTMLを維持するため、`path-note.html`は大きくなる

## 変更時に守ること

- カタログの変更は、保存失敗時に元へ戻せる共通処理を通す
- 外部から読み込むデータは、表示前に検査する
- 画面へ文字列を埋め込む場合は、HTMLエスケープする
- 表示状態を不必要にリセットしない
- 製品ファイルを増やす変更は、配布方法への影響を先に検討する
