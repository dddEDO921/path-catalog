data.jsの階層、各項目の制約、localStorageの用途、読み込み時の検査をまとめます。

01 / DATA.JS

data.jsはJSON互換の代入文

ファイル全体の形
// パスノート data
// ブラウザからの登録・読み込み・
// 書き出しを推奨します。

window.PATH_CATALOG = [
  ...
];
JavaScriptとして読み込めて、代入値はJSONとして安全に解析できる形式です。

読み込み可能な条件

  • 先頭にはコメントを置ける
  • 代入先は window.PATH_CATALOG
  • 代入値はJSONとして解析可能
  • 末尾にセミコロンがある
  • 階層と各項目が厳密な形式を満たす

parseDataJsinspectCatalog strict

02 / HIERARCHY

4段階の配列で分類する

タブ["タブ名", カテゴリ配列]
カテゴリ["カテゴリ名", サブカテゴリ配列]
サブカテゴリ["サブカテゴリ名", パス配列]
パス["表示名", "パス / URL", "ノート"]
順番にも意味がある
配列の順番が、そのままタブ・カテゴリ・サブカテゴリ・パスの表示順になります。
03 / FIELD RULES

各項目の契約

項目読み込み時の条件ブラウザ登録時補足
タブ名空でない文字列必須同名タブの追加は拒否する。
カテゴリ名空でない文字列必須同じタブ内の同名カテゴリ追加は拒否する。
サブカテゴリ名空でない文字列必須通常起動時のみ、空でも警告なしで表示を継続する。
表示名空でない文字列必須一覧と検索結果へ表示する。
パス / URL空でない文字列必須UNC、ドライブパス、URLなどを扱う。
ノート文字列。空文字可任意検索対象に含まれる。
04 / LOCAL STORAGE

ブラウザ内では3種類を保存

キー内容失われた場合
pathCatalog.catalog.v1日常利用の最新カタログ。次回表示時にdata.jsを初期データとして展開する。
pathCatalog.subcatCollapsed.v1閉じているサブカテゴリの組み合わせ。すべて通常の開閉状態へ戻る。
pathNote.theme.v1選択中の表示モード。スタンダード表示へ戻る。
05 / VALIDATION

検査は用途で厳しさが違う

通常起動: 継続を優先

inspectCatalog(catalog)

不完全な行があっても、警告として記録し、表示できるデータは表示します。

ファイル読み込み: 安全を優先

inspectCatalog(catalog, { strict: true })

ブロック形式、3列固定、文字列型、必須項目を満たさない場合は読み込み全体を中止します。

06 / CHANGING THE FORMAT

保存形式を変える前に

data.js

新旧ファイルをどう読み込むか決める。

localStorage

既存利用者の保存データを移行する。

inspectCatalog

通常検査とstrict検査を更新する。

catalogModel

追加・編集・削除処理を更新する。

normalize

画面用オブジェクトへ新項目を渡す。

入出力

書き出し後のdata.jsを再読み込みする。
