01 / HTML

HTML標準機能を使う

HTML

<dialog>

追加、詳細、検索などを、画面上に重ねて表示します。

利点: 開閉状態、Escで閉じる動作、フォーカス制御をブラウザが支援する。
実装: showModal() / close()
HTML

<datalist>

自由入力を残しながら、既存のタブやカテゴリを候補表示します。

利点: 新規入力と既存選択を同じ入力欄で扱える。
実装: addTabOptions / setDatalistOptions
HTML

data-* 属性

動的に生成したボタンや一覧行へ、操作対象や種類を埋め込みます。

利点: 要素ごとに個別のイベント登録をしなくてよい。
実装: data-path-details / data-add-action
ACCESSIBILITY

aria-* 属性

メニューの開閉状態、ショートカット、ボタンの意味を支援技術へ伝えます。

利点: 見た目だけに依存しない操作説明になる。
実装: aria-expanded / aria-controls / aria-keyshortcuts
02 / BROWSER APIS

サーバーなしで機能を作る

STORAGE

localStorage

ブラウザ内へカタログ、表示モード、開閉状態を保存します。

特徴: 文字列だけを保存するため、データはJSONへ変換する。
注意: ブラウザや端末を越えて自動同期されない。
FILE

Blob とObject URL

メモリ上で作ったdata.jsを、ファイルとしてダウンロードします。

流れ: Blob → URL.createObjectURL → 一時リンクをクリック。
後始末: URL.revokeObjectURLでURLを解放する。
CLIPBOARD

Clipboard API

一覧のパスをクリックしてクリップボードへコピーします。

実装: navigator.clipboard.writeText
工夫: 利用できない環境向けの代替コピー処理を持つ。
TIMING

requestAnimationFrame

ダイアログを開いた直後、表示が反映された次の描画タイミングで入力欄へフォーカスします。

理由: 開く処理とフォーカス処理のタイミング競合を避ける。
実装: openSearchDialog
03 / JAVASCRIPT LANGUAGE

よく使うJavaScriptの技

技目的パスノートでの例
配列の map / filter / flatMap元データを直接壊さず、表示や候補に必要な配列を作る。一覧HTML生成、検索、分類候補の収集。
Set重複のない値や、開閉状態の集合を扱う。dedupe、collapsedSubcats。
Map計算済みの値をキーごとに記憶する。カテゴリ色のcolorCache。
Optional chaining ?.途中の値が存在しなくても例外にせず、安全に参照する。catalog?.[t]?.[1]、dialog?.open。
分割代入オブジェクトから必要な値だけを短く取り出す。THEME_CONFIG.map(({ key, label }) => ...)
テンプレートリテラル値を埋め込んだHTMLやdata.js文字列を作る。buildListRow、buildDataJs。
async / awaitファイル読込やコピーなど、完了を待つ処理を順番に書く。importCatalogData、copyTextToClipboard。
try / catch保存や読込の失敗を受け止め、画面へ伝える。commitCatalogChange、localStorage操作。

localStorage: JSONへ変換して保存する
const storage = {
  readJson: (key) => {
    const value = localStorage.getItem(key);
    return value === null ? null : JSON.parse(value);
  },
  writeJson: (key, value) =>
    localStorage.setItem(key, JSON.stringify(value)),
};

storage.writeJson("pathCatalog.catalog.v1", catalog);
注意: localStorageは文字列保存なので、構造化データはJSONへ変換します。
安全なHTML生成: 値をエスケープする
const escapeHtml = (str) =>
  str.replace(/[&<>"']/g, (m) => ({
    "&": "&amp;",
    "<": "&lt;",
    ">": "&gt;",
    '"': "&quot;",
    "'": "&#39;",
  }[m]));

const row = `<span>${escapeHtml(name)}</span>`;
理由: data.jsや入力値をHTMLとして実行させないためです。
data.js読み込み: 実行せずJSONだけ読む
const parseDataJs = (text) => {
  const match = text.match(
    /^\s*window\.PATH_CATALOG\s*=\s*([\s\S]*?)\s*;\s*$/
  );
  if (!match) throw new Error("形式が違います");

  const catalog = JSON.parse(match[1]);
  inspectCatalog(catalog, { strict: true });
  return catalog;
};
ポイント: evalせず、決めた形のJSON部分だけを解析します。
Blob: data.jsを書き出す
const exportData = (catalog) => {
  const text = `window.PATH_CATALOG = ${
    JSON.stringify(catalog, null, 2)
  };\n`;
  const blob = new Blob([text], { type: "text/javascript" });
  const url = URL.createObjectURL(blob);

  const link = document.createElement("a");
  link.href = url;
  link.download = "data.js";
  link.click();
  URL.revokeObjectURL(url);
};
見る場所: buildDataJs、exportCatalogData
build / apply / render: 役割を分ける
const buildListRow = (item) =>
  `<li data-copy="${toHtml(item.path)}">
     ${toHtml(item.name)}
   </li>`;

const applyMain = (html) => {
  $("list").innerHTML = html;
};

const renderMain = () => {
  applyMain(items.map(buildListRow).join(""));
};
利点: HTML生成、DOM反映、再描画の責務が混ざりにくくなります。
05 / IMPLEMENTATION PATTERNS

規模を抑えるための実装パターン

イベント委譲

親要素へ1つだけイベントを登録する。

list.addEventListener("click", ...) と event.target.closest("[data-path-details]")を組み合わせます。再描画で子要素が入れ替わっても、イベントを登録し直す必要がありません。

IIFEで閉じ込める

内部変数を外へ漏らさない。

catalogModel、menuController、noteTooltipは即時実行関数で作られています。外側へ公開する操作だけをreturnします。

build / apply / render

HTML生成とDOM更新を分ける。

build...Htmlは文字列を作り、apply...はDOMへ反映し、render...が両者をまとめます。変更箇所とテスト対象を絞りやすくします。

変更前を複製する

保存失敗時に元へ戻す。

commitCatalogChangeは変更前のcatalogを複製します。変更後のlocalStorage保存に失敗した場合、元のデータへ戻します。

状態から再描画する

DOMを正本にしない。

選択状態はstate、データはcatalogとstate.allへ持ちます。画面はそれらから再生成するため、表示とデータのずれを抑えられます。

06 / SAFETY

安全に扱うための技

XSS

画面へ出す文字をエスケープ

利用者入力やdata.jsの文字列をHTMLとして解釈させません。

実装: toHtml / escapeHtml
原則: 動的な値をテンプレートへ入れる前に変換する。
IMPORT

evalを使わずJSON解析

data.jsをJavaScriptとして実行せず、決められた代入文からJSON部分だけを取り出します。

実装: 正規表現 → JSON.parse → inspectCatalog
効果: 読み込んだファイル内の任意コードを実行しない。
VALIDATION

外部データを検査する

配列構造、文字列型、必須項目を確認してから利用します。

通常起動: 警告しながら表示を継続。
インポート: strict検査で不正なら全体を拒否。
NORMALIZATION

文字列とデータ形を整える

未定義や表記ゆれを、後続処理が扱いやすい形へ変換します。

実装: toSafeString / normalize
検索: Unicode NFKC正規化と小文字化を使う。
07 / WHAT IS NOT USED

あえて使っていないもの

フロントエンドフレームワーク

ReactやVueは使っていません。ビルド不要でHTMLを開くだけで動かすことを優先しています。

バックエンド・データベース

サーバーへ送信せず、ブラウザ内のlocalStorageとdata.jsで完結します。

モジュール分割

製品のJavaScriptとCSSはHTML内に置き、配布する製品を2ファイルに保っています。

任意JavaScriptの実行

インポートしたdata.jsは実行せず、JSON互換部分だけを安全に解析します。
