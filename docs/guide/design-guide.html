DESIGN PHILOSOPHY

思い出すための、
静かな道具。

パスノートは、パスを美しく管理するための画面ではありません。「あれ、どこだっけ」を短くするための道具です。デザインは、整理の正しさよりも、発見の速さと使い続けやすさを支えます。

01 / CORE

中心思想

分類させるより、思い出させる。
利用者は、正しいカテゴリ名を覚えているとは限りません。だから、分類・検索・文脈追加のどこからでも目的のパスへ近づける設計にします。
02 / PRINCIPLES TO UI

思想と表現

探す前に迷わせない

入口を明確にする。

表現: 検索はヘッダーに常設。カテゴリはサイドバーに固定。追加は「＋ 追加」に集約。

判断: 新しい操作を増やすとき、入口が散らばるならメニューへまとめる。

分類は主役にしない

主役はパス。

表現: カテゴリは色線や見出しで控えめに示し、一覧ではパス名とパス文字列を優先する。

判断: 分類の装飾がパスより目立つなら弱める。

文脈があるなら選ばせない

今いる場所を使う。

表現: サブカテゴリ追加やパス追加は、そのカテゴリ・サブカテゴリの近くに置く。

判断: 追加先が画面上で明らかなのに、プルダウンで再選択させない。

整理と登録を混ぜない

目的を分ける。

表現: タブ・カテゴリの追加と、パス登録を別の入口にする。空のパス行を先に作らない。

判断: 「分類を整えたいだけ」の人に、パス入力を要求しない。

密度は選べるべき

使い方は一つではない。

表現: スタンダード、オープン、コンパクトの3表示を持つ。

判断: 表示モードは色違いではなく、読み方や情報密度の違いを持たせる。

操作の反応は静かでよい

作業を止めない。

表現: コピーや保存は軽いトースト。詳細・編集だけダイアログで集中させる。

判断: 成功通知のためにモーダルを出さない。

03 / VIEW MODES

表示モードの思想

スタンダード

構造をつかむ

初見向け。カードとまとまりで、タブ・カテゴリ・サブカテゴリの階層を理解しやすくする。

オープン

軽く読む

カードを減らし、見出しと線で流し読みする。情報の圧を下げる。

大量に見る

表に近い密度で、件数が多いときの比較とスキャンを優先する。

04 / DO NOT

避けること

装飾が主役

色、影、アイコンがパスより目立つ。

入口が多すぎる

便利そうなボタンを画面中に置く。

分類を強制する

きれいに分類しないと登録できない。

色違いの表示モード

密度や読み方の差がない。

説明前提の操作

READMEを読まないと分からない。

管理者目線だけ

日々探す人の行動を軽視する。
