DESIGN PHILOSOPHY

UIは、考える量を
減らすためにある。

パスノートのデザインは、見た目を整えるためだけのものではありません。利用者が覚える、迷う、探す、入力する負担を減らすための設計です。

01 / CORE

中心思想

記憶より認識。操作より発見。
ユーザーに正しい場所や手順を思い出させるのではなく、画面を見れば次に取れる行動が分かる状態を目指します。
02 / PRINCIPLES TO UI

原則と表現

認識できること

RecallよりRecognition。

考え方: ユーザーに場所や名前を正確に思い出させない。見れば候補や現在地が分かるようにする。

このツールでの表現: カテゴリをサイドバーに固定し、検索をヘッダーに常設する。タブ・カテゴリ・サブカテゴリの階層を画面上に見せる。

入口が明確であること

迷わせない。

考え方: 重要な操作は探させない。複数の操作がある場合は、入口を整理して選ばせる。

このツールでの表現: 全体の追加操作は「＋ 追加」に集約する。検索、表示、データ管理はヘッダーへまとめる。

文脈を保つこと

今いる場所を使う。

考え方: 画面上で対象が明らかな操作では、同じ情報を再入力・再選択させない。

このツールでの表現: サブカテゴリ追加やパス追加は、対象カテゴリの近くに置く。追加先が決まっている場合は入力欄をロックする。

段階的に開示すること

最初から全部見せない。

考え方: 普段必要な情報を優先し、編集・削除・詳細確認のような重い操作は必要になったときだけ出す。

このツールでの表現: 一覧ではコピーと識別を優先する。詳細・編集・削除は詳細ダイアログに分ける。

作業を止めないこと

軽い反応にする。

考え方: 成功やコピーのたびに大きな確認を出さない。流れを止める通知は、本当に判断が必要なときに限る。

このツールでの表現: コピーや保存はトーストで軽く伝える。削除やインポートのような破壊的操作だけ確認を出す。

密度を選べること

使い方に合わせる。

考え方: 初見で理解しやすい画面と、大量データを素早く見る画面は同じではない。ユーザーの状況に合わせて表示密度を変えられるべき。

このツールでの表現: スタンダード、オープン、コンパクトの3表示を用意する。単なる色違いではなく、読み方と密度を変える。

03 / VIEW MODES

表示モードの意味

スタンダード

理解する

カードとまとまりで構造を把握する。初見や少量データ向け。

オープン

読む

装飾を減らし、見出しと線で軽く読む。情報の圧を下げる。

走査する

表に近い密度で、大量のパスを比較・スキャンする。

04 / DECISION CHECK

判断基準

認識できるか

ユーザーに覚えさせていないか。

入口は明確か

重要操作が散らばっていないか。

文脈を保てるか

今いる場所を再選択させていないか。

段階的か

最初から重い情報を見せていないか。

流れを止めないか

不要な確認や通知を出していないか。

密度を選べるか

少量・大量のどちらかに偏っていないか。

05 / DO NOT

避けること

分類を強制する

正しい分類を知らないと使えない。

入口を増やしすぎる

便利そうなボタンが画面全体を騒がせる。

装飾が主役になる

色や影がパスより目立つ。

空の登録を作る

あとで埋める前提の空データを増やす。

色違いだけのモード

表示モードに役割の違いがない。

説明を前提にする

読まないと使えないUIにする。
