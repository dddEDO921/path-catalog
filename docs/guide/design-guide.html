DESIGN GUIDE

見た目は、
操作の迷いを減らすためにある。

パスノートのCSS構成、表示モード、UI/UX判断、色や余白の使い方をまとめます。デザイン変更時の判断基準として使います。

01 / DESIGN PRINCIPLES

デザインの基本方針

探すことを邪魔しない

主役はパス名とパス文字列です。装飾は視線誘導のために使い、情報量を増やしすぎません。

追加操作は文脈に置く

サブカテゴリやパス追加は、追加したい場所の近くに置きます。全体追加はヘッダーのメニューに集約します。

表示密度を選べる

使う人やデータ量に合わせて、スタンダード・オープン・コンパクトを切り替えられるようにします。

製品は2ファイルのまま

CSSはHTML内に残します。大きさの代償は、テーマ変数、章コメント、資料で補います。

02 / CSS ARCHITECTURE

CSSは役割で読む

章役割代表例
テーマ変数色、角丸、影、パス表示などの基本値を定義する。--bg-base、--accent、--card-radius
共通UIタブ、カテゴリ、一覧行、詳細ボタンなど、表示モードをまたぐ部品。.tab-btn、.cat-btn、.list-row
操作UIメニュー、ダイアログ、ボタン、通知など、操作のための部品。.add-menu、.dialog-shell、.primary-button
表示モードスタンダード、オープン、コンパクト固有の見た目。[data-theme="open"]、.compact-list
レスポンシブ狭い画面で操作しやすい形へ調整する。@media (max-width: 560px)
読む順番
まずテーマ変数で全体の色と形を見て、次に共通UI、最後に表示モード固有の上書きを見ます。
03 / DESIGN TOKENS

色と形は変数で管理する

--bg-base画面の土台。スタンダードは暗色。
--bg-layerヘッダーやカードの面。
--accent選択、主要ボタン、強調線。
--text-body本文の標準色。
:root {
  --bg-base: #0f172a;
  --bg-layer: #111827;
  --accent: #7c3aed;
  --text-heading: #e5e7eb;
  --card-radius: 0.75rem;
  --card-shadow: 0 8px 24px -12px rgba(0, 0, 0, 0.8);
}
04 / VIEW MODES

3つの表示モード

スタンダード

スタンダード

カード、影、暗色背景でまとまりを作る。通常利用の基準。

data-theme="current"

オープン

オープン

カード感を減らし、見出しと罫線で軽く見せる。読み流しやすさ重視。

data-theme="open"

コンパクト

表に近い密度で、大量のパスを一覧する。情報量重視。

data-theme="compact"

05 / COMPONENTS

主要コンポーネントの意図

TAB

タブ

常に1つ選択される大分類です。線と文字色で選択状態を示し、ボタンを増やしすぎないようにします。

CSS: .tab-btn / .tab-btn.active
CATEGORY

カテゴリ

サイドバーで現在の絞り込みを示します。色付きラインでカテゴリ色を認識しやすくします。

CSS: .cat-btn / .cat-line
PATH

パス行

クリックでコピーできる行です。ホバー時に少しだけ動かし、操作できることを示します。

CSS: .list-row / .path-chip
DIALOG

ダイアログ

追加・編集・検索を画面遷移なしで行います。背景をぼかし、今の作業に集中させます。

CSS: .dialog-shell / dialog::backdrop
06 / UI/UX DECISIONS

UI/UXの判断

汎用追加メニュー

ヘッダーに集約する。

タブ、カテゴリ、パス追加を1つの「追加」メニューにまとめています。画面上へプラスボタンを増やしすぎず、初めての人でも入口を見つけやすくします。

文脈内の追加ボタン

必要な場所にだけ残す。

サブカテゴリやパスは、追加したいカテゴリの近くにボタンを置きます。プルダウンで選ばせるより、今見ている場所へ追加する方が自然です。

空登録を避ける

選んでから入力する。

タブ整理とパス登録は別のユースケースです。すべてを一度に入力させず、まず何を追加するか選ばせます。

検索は横断的に

場所を覚えていなくても探せる。

名前、パス、ノート、分類を対象にします。パスノートの本質は「場所を思い出す」ことなので、分類をたどる以外の入口を用意します。

07 / CSS SAMPLES

サンプルコード

テーマ変数で見た目を切り替える
:root,
[data-theme="current"] {
  --bg-base: #0f172a;
  --accent: #7c3aed;
  --card-radius: 0.75rem;
}

[data-theme="open"] {
  --bg-base: #f4f4f5;
  --accent: #4f477b;
  --card-radius: 0;
}
部品側は変数を見るだけなので、表示モードごとの上書きを小さくできます。
状態はクラスと属性で表す
.tab-btn {
  border-bottom: 2px solid transparent;
  color: var(--text-muted);
}

.tab-btn.active {
  border-bottom-color: var(--accent);
  color: var(--text-heading);
}

.subcat-toggle[aria-expanded="false"] .subcat-line {
  width: 6px;
  opacity: 0.35;
}
選択状態は.active、開閉状態はaria-expandedに寄せます。
08 / CHANGE CHECKLIST

デザイン変更時の確認

3表示モード

スタンダード、オープン、コンパクトで破綻しない。

状態

hover、active、disabled、dialog openを確認する。

密度

大量のパスでも読みづらくならない。

モバイル

560px以下で横にはみ出さない。

操作性

追加・検索・コピーが見つけやすい。

資料

CSS構成やUI判断が変わったらこの資料も更新する。
