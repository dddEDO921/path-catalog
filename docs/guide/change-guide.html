入門手順実装

実現したいことから入口を探し、データ・表示・操作への影響を整理してから実装します。

01 / START FROM YOUR GOAL

変更したいことを選ぶ

02 / SMALL TUTORIAL

小さな変更を最後まで追う

例として「検索結果にノートを表示する」を考えます。保存形式を変えず、表示だけを変える変更です。

1

入口を探す

検索結果の見た目を作る関数を特定します。

buildSearchResultHtml
2

データを確認

引数のitemには、すでにノートがあります。

item.note
3

安全に表示する

HTML文字列へ追加し、安全な文字列へ変換します。

${toHtml(item.note)}
4

影響を確認する

空・長いノート・大量結果を確認します。

renderSearchResults
変更不要

data.jsとlocalStorage。

変更する

buildSearchResultHtmlと必要なCSS。

確認する

検索、詳細、モバイル幅、長い文字列。

03 / BEFORE CODING

実装前に答えること

データは変わるか

保存形式が変わるなら、既存localStorageと過去のdata.jsの互換性を決めます。

どの操作から始まるか

クリック、フォーム送信、キー操作など、入口となるイベントを特定します。

どの表示へ影響するか

スタンダード・オープン・コンパクト、検索、詳細画面を確認します。

失敗したらどうなるか

保存失敗、壊れたデータ、対象が見つからない場合を考えます。

04 / DONE CHECK

完了の基準

構文

インラインJavaScriptの構文エラーがない。

主要操作

対象機能と近接する操作が動く。

画面幅

PCとモバイルで崩れない。

データ

再読込・書き出し・読み込み後も保たれる。

失敗時

エラーが利用者へ伝わり、データを壊さない。

資料

設計や操作が変わった場合は対応資料も直す。
