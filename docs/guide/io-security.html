JSONだけ解析

data.jsはJavaScriptファイルですが、インポート時にはJavaScriptとして実行しません。window.PATH_CATALOG = ...;の形だけを受け取り、右辺をJSON.parseします。
解析の考え方
const parseDataJs = (text) => {
  const match = text.match(
    /^\s*window\.PATH_CATALOG\s*=\s*([\s\S]*?)\s*;\s*$/
  );
  if (!match) throw new Error("形式が違います");

  return JSON.parse(match[1]);
};
実コードではコメントとBOMにも対応し、strict検査まで行います。
読み込み後の確認
const imported = parseDataJs(text);
inspectCatalog(imported, { strict: true });

if (confirm("現在のデータを置き換えますか？")) {
  commitCatalogChange(() => {
    catalog = cloneCatalog(imported);
  });
}
壊れたデータや意図しない置き換えを防ぎます。
04 / HTML OUTPUT

画面へ出す前にエスケープする

危険な例
main.innerHTML = `
  <span>${item.name}</span>
`;
入力値にHTMLが含まれると、DOMとして解釈されます。
安全な例
main.innerHTML = `
  <span>${toHtml(item.name)}</span>
`;
動的な文字列はtoHtmlを通してからHTMLへ入れます。
テンプレートリテラルは便利だが境界が曖昧になりやすい
HTMLの形を作る場所では、固定のタグと利用者由来の値を分けて考えます。値は表示直前に必ずエスケープします。

06 / SECURITY CHECKLIST

変更時の確認項目

外部入力

ファイル、localStorage、フォーム入力をそのまま信用していないか。

HTML出力

利用者由来の値をtoHtmlなしでinnerHTMLへ入れていないか。

インポート

読み込んだdata.jsを実行していないか。

検査

strict検査が必要な境界で、通常検査だけにしていないか。

保存失敗

変更前へ戻せる構造になっているか。

破壊的操作

削除・インポート前に利用者へ確認しているか。
