03 / MAIN FLOWS

よく追う処理の流れ

パスを追加する

onAddPathSubmit → catalogModel.appendItem → commitCatalogChange → refreshCatalog

一覧を表示する

renderMain → buildMainHtml → buildListRow → applyMain

横断検索する

renderSearchResults → getSearchResults → buildSearchResultHtml

データを読み込む

parseDataJs → inspectCatalog → saveCatalog → refreshCatalog

04 / GLOSSARY

頻出する名前

catalog

保存・編集用の階層データ。

state.all

検索・表示用の平坦なパス一覧。

catalogModel

階層データを変更する窓口。

commitCatalogChange

変更、保存、復元、再描画をまとめる処理。

inspectCatalog

外部データの形を検査する処理。

normalize

検査済みデータを画面用へ整える処理。

build...Html

表示するHTML文字列を作る処理。

renderAll

主要な画面をまとめて描き直す処理。
