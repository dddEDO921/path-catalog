STATE MANAGEMENT

何を正本にするかを
先に決める。

パスノートの状態管理は、保存用データ、画面用データ、表示設定、一時的なUI状態を分けて考えます。混ぜると、編集後の表示や検索、削除後の再描画が壊れやすくなります。

01 / FOUR STATE TYPES

状態は4種類ある

種類主な変数・保存先役割変更方法
保存用データcatalogタブ、カテゴリ、サブカテゴリ、パスの正本。catalogModel経由で変更する。
画面用データstate.all検索、絞り込み、一覧表示のための平坦な配列。refreshCatalogで作り直す。
表示状態state.tab、state.cat、表示モード今どこを見ているか、どう表示するか。setSelection、applyThemeで変える。
一時UI状態ダイアログ、メニュー、詳細編集中の項目その場限りの操作状態。専用関数で開閉・更新する。
02 / SOURCE OF TRUTH

正本のルール

catalogがデータの正本

タブやカテゴリの空グループ、表示順、パスの追加・編集・削除はcatalogが持ちます。localStorageは永続化先であり、実行中に直接参照し続ける正本ではありません。

state.allは派生データ

catalogから作る検索・表示用の配列です。ここだけを書き換えても保存データは変わりません。データ変更後は必ず作り直します。

DOMは正本ではない
画面上のHTMLからデータを復元しようとしません。DOMは最後に描かれた結果であり、次の描画で置き換えられるものです。
03 / DATA SHAPES

保存用と画面用の違い

保存用 catalog
["共通", [
  ["申請", [
    ["経費", [
      ["経費精算フォルダ", "\\\\server\\expense", ""]
    ]]
  ]]
]]
空の分類と順番を保持できます。
画面用 state.all
{
  id: 12,
  tab: "共通",
  category: "申請",
  subCategory: "経費",
  name: "経費精算フォルダ",
  path: "\\\\server\\expense",
  _meta: { t: 0, c: 0, s: 0, i: 0 }
}
検索、コピー、詳細表示、編集対象の特定が簡単になります。
04 / SELECTION STATE

選択状態は表示の文脈

TAB

state.tab

常に1つ選択されます。カテゴリ追加は、この選択中タブを前提にできます。

CATEGORY

state.cat

空文字なら「すべて」です。編集後に勝手に絞り込みを変えないため、選択状態は明示的に扱います。

SUBCATEGORY

近接操作の文脈

サブカテゴリ追加やパス追加は、押した場所からカテゴリ・サブカテゴリを受け取ります。

THEME

表示モード

スタンダード、オープン、コンパクトは同じデータを別の密度で見る状態です。

05 / UPDATE RULES

状態を変えるときのルール

データ変更

catalogModelで変更し、commitCatalogChangeへ渡す。

画面用データ

state.allを直接編集せず、refreshCatalogで作り直す。

選択変更

setSelectionを通して、タブ・カテゴリの整合性を保つ。

表示モード

applyThemeでDOM属性とlocalStorageを同期する。

開閉状態

サブカテゴリの開閉は専用Setで扱い、localStorageへ保存する。

一時状態

詳細ダイアログや編集中項目は、閉じるときにリセットする。

06 / COMMON MISTAKES

壊れやすい変更

state.allだけを書き換える

画面は変わったように見えても、保存・書き出し・再読込で消えます。保存したい変更は必ずcatalogへ入れます。

DOMから値を拾いすぎる

一覧は表示モードで形が変わります。対象特定にはdata-*属性や_metaを使います。

保存後に選択状態を変える

編集後に勝手にカテゴリを絞ると、ユーザーの前の文脈が失われます。必要なときだけ明示的に変えます。

localStorageを直接読み続ける

起動時に読み、実行中はメモリ上のcatalogとstateを使います。保存は変更確定時に行います。
